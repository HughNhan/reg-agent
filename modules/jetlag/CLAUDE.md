# Phase 2: Jetlag Cluster Deployment Module

## Purpose

Deploy OpenShift clusters on bare metal using Jetlag's Ansible automation and Assisted Installer.

## Module Scope

**Focus**: OpenShift cluster deployment only (no import)
**Input**: CLOUD_NAME, LAB from Phase 1; cluster config from `vars/config.env`
**Output**: BASTION_HOST, KUBECONFIG_PATH, CLUSTER_TYPE in `vars/state.env`
**Next Phase**: Crucible installation (Phase 3)

**Note**: This module only deploys clusters. If you have an existing cluster, use `cluster-ready` mode and provide BASTION_HOST/KUBECONFIG_PATH directly in config.

## Key Script

### `jetlag-deploy.sh`

**Location**: `modules/jetlag/jetlag-deploy.sh`
**Purpose**: Deploy new OpenShift cluster using Jetlag
**Called by**: `make deploy` or top-level `make deploy` in full mode

#### Dependencies Checked

1. **Repository**: jetlag exists at `repos/jetlag/`
2. **State variables** (from Phase 1):
   - CLOUD_NAME
   - LAB
3. **Configuration variables**:
   - CLUSTER_TYPE (mno or sno)
   - OCP_BUILD (ga, dev, ci)
   - OCP_VERSION (e.g., latest-4.20)
   - NETWORK_STACK (ipv4, ipv6, dual)
   - WORKER_NODE_COUNT (for MNO)
4. **Files**: Pull secret at `pull-secret.txt` or `$PULL_SECRET_PATH`
5. **Commands**: ansible-playbook, sed, awk, grep

#### What It Does

1. **Dependency Check**: Validates prerequisites
2. **Generate all.yml**: Creates Jetlag configuration:
   ```yaml
   lab: scalelab
   lab_cloud: cloud42
   cluster_type: mno
   worker_node_count: 3
   ocp_build: "ga"
   ocp_version: "latest-4.20"
   ```
3. **Create Inventory**: Runs `create-inventory.yml` to generate `ansible/inventory/cloud42.local`
4. **Extract Bastion**: Parses inventory to find bastion hostname
5. **Setup Bastion**: Runs `setup-bastion.yml` (installs Assisted Installer, DNS, registry)
6. **Deploy Cluster**: Runs `mno-deploy.yml` or `sno-deploy.yml`
7. **Save State**: Writes BASTION_HOST, KUBECONFIG_PATH to `vars/state.env`

## Configuration Variables

### Required (from `vars/config.env`):
- `CLUSTER_TYPE`: "mno" or "sno"
- `OCP_BUILD`: "ga", "dev", or "ci"
- `OCP_VERSION`: e.g., "latest-4.20", "4.18.1"
- `NETWORK_STACK`: "ipv4", "ipv6", or "dual"
- `PULL_SECRET_PATH`: Path to OpenShift pull secret
- `WORKER_NODE_COUNT`: Number of workers (MNO only)

### Optional:
- `IPV6_MODE`: "proxy" or "disconnected" (when NETWORK_STACK=ipv6)
- `BMC_PASSWORD`: Custom BMC password
- `BASTION_ROOT_PASSWORD`: For automatic ssh-copy-id

## Output State

Written to `vars/state.env`:

```bash
BASTION_HOST=cloud42-h01-000-r750.scalelab.example.com
KUBECONFIG_PATH=/root/mno/kubeconfig
CLUSTER_TYPE=mno
```

## Testing This Module

```bash
# Prerequisites: Phase 1 must have run successfully
cat vars/state.env
# Should show: CLOUD_NAME, LAB

# Initialize Jetlag
cd modules/jetlag
make init

# Deploy cluster
make deploy

# Verify output
cat ../../vars/state.env
# Should now show: BASTION_HOST, KUBECONFIG_PATH, CLUSTER_TYPE

# Validate cluster
make validate

# Manual verification
ssh root@$(grep BASTION_HOST ../../vars/state.env | cut -d= -f2)
oc --kubeconfig=/root/mno/kubeconfig get nodes
```

## Common Issues

### Issue 1: "State file not found"
**Cause**: Phase 1 didn't run or didn't complete successfully
**Fix**: Run Phase 1 first: `make test-quads`

### Issue 2: "Pull secret file not found"
**Cause**: No pull secret at expected location
**Fix**:
```bash
# Place pull secret at repo root
cp ~/pull-secret.txt /path/to/reg-agent/pull-secret.txt

# Or set custom path in vars/config.env
echo "PULL_SECRET_PATH=/path/to/pull-secret.txt" >> vars/config.env
```

### Issue 3: "Inventory not created"
**Cause**: Jetlag create-inventory failed (invalid CLOUD_NAME, lab issues)
**Debug**:
```bash
cd repos/jetlag
source .ansible/bin/activate
ansible-playbook ansible/create-inventory.yml -vv
```

### Issue 4: "Cannot SSH to bastion"
**Cause**: SSH keys not copied to bastion
**Fix**:
```bash
# Get bastion host from state
BASTION=$(grep BASTION_HOST vars/state.env | cut -d= -f2)

# Copy SSH key manually
ssh-copy-id root@$BASTION
```

### Issue 5: "Cluster deployment failed"
**Common causes**:
- Insufficient bare metal machines
- BMC access issues
- Network connectivity problems
- OpenShift version not available

**Debug**:
```bash
# Check Jetlag logs
cd repos/jetlag
tail -100 /root/assisted-installer.log  # On bastion
```

## Jetlag-Specific Details

### Cluster Types

**MNO (Multi-Node OpenShift)**:
- 3 control-plane nodes
- N worker nodes (configurable)
- Typical for performance testing

**SNO (Single-Node OpenShift)**:
- 1 node per cluster
- Creates multiple SNO clusters from available machines

### Network Configurations

**IPv4** (default):
- Simple, no registry or proxy needed
- Most common for testing

**IPv6**:
- Requires either proxy or disconnected mode
- Disconnected = local registry with mirrored content
- Proxy = bastion proxy for external access

**Dual Stack**:
- Both IPv4 and IPv6
- No special registry/proxy needed

### Bastion Configuration

Jetlag's `setup-bastion.yml` installs:
- Assisted Installer service
- DNS server (dnsmasq)
- HTTP server for ISO/ignition files
- Optional: Local registry (for disconnected)
- Optional: Squid proxy (for IPv6 proxy mode)

## Development Tips

### Modifying OCP Version

Edit `vars/config.env`:
```bash
# Use latest stable
OCP_VERSION=latest-4.20

# Use specific version
OCP_VERSION=4.18.1

# Use latest development build
OCP_BUILD=dev
OCP_VERSION=latest-4.21
```

### Adding Custom Jetlag Variables

Edit `modules/phase-2-jetlag-deploy.sh` after generating `all.yml`:
```bash
# Add custom variable
echo "my_custom_var: value" >> repos/jetlag/ansible/vars/all.yml
```

### Debugging Inventory Generation

```bash
cd repos/jetlag
source .ansible/bin/activate

# View generated inventory
cat ansible/inventory/cloud42.local

# Manually regenerate
ansible-playbook ansible/create-inventory.yml
```
