# Phases 4-6: Regulus Testing Module

## Purpose

Configure Regulus performance test framework, execute tests, and validate results.

## Module Scope

**Focus**: Regulus setup, test execution, and result validation
**Input**: BASTION_HOST, KUBECONFIG_PATH from Phase 2; Crucible from Phase 3
**Output**: Test results, validation reports in `artifacts/`
**Previous Phases**: QUADS (Phase 1), Jetlag (Phase 2), Crucible (Phase 3)

## Files in This Module

```
modules/
├── phase-4-regulus-setup.sh       # Regulus configuration and setup
├── phase-5-regulus-run.sh         # Test execution
└── phase-6-validate-results.sh    # Result validation

repos/cpt-regulus/                  # External dependency
├── run_cpt.sh                     # Main test runner
├── bootstrap.sh                   # Environment setup
├── bin/
│   ├── reg-smart-config           # Auto-detect NICs
│   └── ...                        # Other utilities
├── lab.config                     # Generated by Phase 4
├── jobs.config                    # Generated by Phase 4
└── [job directories]              # Test job definitions
    ├── 1_GROUP/
    │   └── NO-PAO/
    │       ├── 4IP/               # Basic IP tests
    │       └── ...
    └── 2_GROUP/
        └── PAO/                   # Performance optimized tests
```

## Phase 4: Regulus Setup

### Script: `phase-4-regulus-setup.sh`

**Purpose**: Configure Regulus on bastion with auto-generated configs
**Called by**: `Makefile` target `test-regulus-install` or `deploy-*`

#### Dependencies Checked

1. **Repository**: cpt-regulus exists at `repos/cpt-regulus/`
2. **State variables** (from Phase 2):
   - BASTION_HOST
   - KUBECONFIG_PATH
3. **Configuration variables**:
   - REGULUS_TEST (test job path)
4. **Remote dependencies**:
   - Crucible installed at `/root/crucible` on bastion
5. **SSH access**: Passwordless SSH to bastion
6. **Commands**: ssh, scp, tar (or rsync)

#### What It Does

1. **Clone Regulus to Bastion**:
   - Creates timestamped directory: `/root/cpt-regulus-<datetime>`
   - Clones from https://github.com/HughNhan/regulus.git
   - Creates `latest` symlink pointing to this installation

2. **Generate `lab.config`** (initial template):
   ```bash
   export REG_KNI_USER=kni
   export REG_OCPHOST=${BASTION_HOST}
   export KUBECONFIG=${KUBECONFIG_PATH}

   # Worker nodes (will be auto-detected by reg-smart-config in Phase 5)
   export OCP_WORKER_0=
   export OCP_WORKER_1=
   export OCP_WORKER_2=

   # NIC configuration (will be auto-detected by reg-smart-config in Phase 5)
   export REG_SRIOV_NIC=
   export REG_SRIOV_NIC_MODEL=
   export REG_MACVLAN_NIC=
   export REG_DPDK_NIC=
   ```

3. **Generate `jobs.config`**:
   ```bash
   export OCP_PROJECT=crucible-regulus
   export JOBS=${REGULUS_TEST}  # e.g., ./1_GROUP/NO-PAO/4IP/INTER-NODE/TCP/2-POD
   export TAG=${REGULUS_TAG:-REG-AGENT}
   export NUM_SAMPLES=${NUM_SAMPLES:-3}
   export DURATION=${TEST_DURATION:-60}
   ```

4. **Bootstrap Regulus Environment**:
   - Runs `bootstrap.sh` to setup Python venv and environment
   - Detects cluster type (SNO, Compact MNO, Standard MNO)
   - Fixes `worker_labels.config` for SNO/Compact clusters

5. **Save State**:
   ```bash
   REGULUS_PATH=/root/cpt-regulus
   ```

#### Configuration Variables

**Required** (from `vars/config.env`):
- `REGULUS_TEST`: Test job directory path (e.g., `./1_GROUP/NO-PAO/4IP`, `./1_GROUP/NO-PAO/4IP/INTER-NODE/TCP/2-POD`)

**Optional**:
- `REG_KNI_USER`: SSH user for cluster nodes (default: `kni`)
- `REGULUS_TAG`: Test identifier tag (default: `REG-AGENT`)
- `NUM_SAMPLES`: Number of test samples (default: `3`)
- `TEST_DURATION`: Test duration in seconds (default: `60`)
- `REG_DP`: Deployment identifier (default: `reg-agent`)

#### Output State

Written to `vars/state.env`:
```bash
REGULUS_PATH=/root/cpt-regulus
```

## Phase 5: Test Execution

### Script: `phase-5-regulus-run.sh`

**Purpose**: Execute tests via `run_cpt.sh` and collect results
**Called by**: `Makefile` target `test` or `deploy-*`

#### What It Does

1. **Validate Prerequisites**:
   - Check SSH access to bastion
   - Verify `run_cpt.sh` exists at `${REGULUS_PATH}`

2. **Create Run Artifacts**:
   ```bash
   RUN_ID=run-$(date +%Y%m%d-%H%M%S)
   ARTIFACT_DIR=${REG_AGENT_ROOT}/artifacts/${RUN_ID}
   ```

3. **Execute Tests on Bastion** (via `run_cpt.sh`):
   ```bash
   ssh root@$BASTION_HOST bash <<'REGULUS_RUN'
   cd ${REGULUS_PATH}
   source bootstrap.sh
   ./run_cpt.sh  # Runs: reg-smart-config → init-lab → init-jobs → tests
   REGULUS_RUN
   ```

   **Note**: `run_cpt.sh` internally handles:
   - `reg-smart-config`: Auto-detect NICs and workers
   - `make init-lab`: Run lab-analyzer, SRIOV_INIT, INVENTORY
   - `make init-jobs`: Configure test jobs
   - Test execution

4. **Collect Results**:
   - Reads symlink at `${REGULUS_PATH}/latest` (points to latest results)
   - Copies results from bastion to local `artifacts/${RUN_ID}/regulus-results/`
   - Copies execution log to `artifacts/${RUN_ID}/logs/regulus-run.log`

5. **Create Symlink**:
   ```bash
   artifacts/latest -> ${RUN_ID}
   ```

6. **Save Run Metadata**:
   ```bash
   RUN_ID=run-20260609-143022
   RUN_EXIT_CODE=0
   RUN_TIMESTAMP=2026-06-09T14:30:22-05:00
   ```

#### Output Files

**Artifacts Structure**:
```
artifacts/
├── run-20260609-143022/           # Timestamped run directory
│   ├── logs/
│   │   └── regulus-run.log        # Full execution log
│   └── regulus-results/           # Copied from bastion
│       ├── result-summary.txt     # Test result summary
│       ├── crucible-data/         # Raw Crucible data
│       └── ...                    # Additional result files
└── latest -> run-20260609-143022  # Symlink to latest run
```

#### Exit Codes

- `0`: Tests completed successfully
- Non-zero: Tests failed (exit code from `run_cpt.sh`)

## Phase 6: Result Validation

### Script: `phase-6-validate-results.sh`

**Purpose**: Validate test results and generate validation report
**Called by**: `Makefile` target `test-run` or `deploy-*` (runs after Phase 5)

#### What It Does

1. **Locate Results**:
   - Uses `RUN_ID` from state to find artifact directory
   - Validates artifact directory exists

2. **Run Validation Checks**:

   **Check 1: Test Execution Status**
   - Examines `RUN_EXIT_CODE` from state
   - PASS if exit code = 0, FAIL otherwise

   **Check 2: Result Files Exist**
   - Verifies `regulus-results/` directory exists
   - Counts result files
   - Checks for key files like `result-summary.txt`

   **Check 3: Parse Result Summary**
   - Reads `result-summary.txt` if present
   - Extracts performance metrics
   - Includes in validation report

   **Check 4: Log Analysis**
   - Verifies execution log exists
   - Checks log size
   - Scans for error mentions (warning only)

3. **Generate Reports**:

   **Validation Report** (`validation-report.txt`):
   ```
   =========================================
   reg-agent Validation Report
   =========================================
   Run ID: run-20260609-143022
   Timestamp: 2026-06-09 14:35:10

   Check 1: Test Execution Status
   --------------------------------------
   ✅ Test execution: PASSED (exit code 0)

   Check 2: Result Files
   --------------------------------------
   ✅ Result files: FOUND (127 files)
     ✓ result-summary.txt found

   Check 3: Result Summary
   --------------------------------------
   Result Summary:
   [Contents of result-summary.txt]
   ✅ Metrics found in result summary

   Check 4: Logs
   --------------------------------------
   ✅ Execution log: FOUND (524288 bytes)

   =========================================
   Overall Validation Result
   =========================================
   ✅ VALIDATION PASSED
   ```

   **Validation Summary JSON** (`validation-summary.json`):
   ```json
   {
     "run_id": "run-20260609-143022",
     "timestamp": "2026-06-09T14:35:10-05:00",
     "validation_status": "PASSED",
     "test_exit_code": 0,
     "artifacts_dir": "/path/to/reg-agent/artifacts/run-20260609-143022"
   }
   ```

4. **Save Validation Status**:
   ```bash
   VALIDATION_STATUS=PASSED  # or FAILED
   ```

#### Output Files

```
artifacts/${RUN_ID}/
├── validation-report.txt      # Human-readable validation report
└── validation-summary.json    # Machine-readable summary
```

#### Exit Codes

- `0`: Validation PASSED
- `1`: Validation FAILED

## Common Issues

### Phase 4 Issues

#### Issue 1: "Crucible installation not found"
**Cause**: Phase 3 didn't complete or Crucible not at expected path
**Fix**:
```bash
# Run Phase 3
make test-crucible

# Verify Crucible exists
ssh root@$BASTION_HOST "ls -la /root/crucible"
```

#### Issue 2: "reg-smart-config had issues"
**Impact**: NICs not auto-detected, manual configuration needed
**Fix**:
```bash
# SSH to bastion
ssh root@$BASTION_HOST
cd /root/cpt-regulus

# Manually edit lab.config
vi lab.config

# Set NIC variables manually
export REG_SRIOV_NIC=ens2f0
export REG_MACVLAN_NIC=ens2f1
export OCP_WORKER_0=worker-0.cluster.example.com
# etc.
```

#### Issue 3: "Cannot determine worker nodes"
**Cause**: Cluster not accessible or node discovery failed
**Debug**:
```bash
ssh root@$BASTION_HOST
export KUBECONFIG=/root/mno/kubeconfig
oc get nodes
oc get nodes -l node-role.kubernetes.io/worker
```

### Phase 5 Issues

#### Issue 4: "run_cpt.sh not found"
**Cause**: Regulus not properly copied or wrong path
**Fix**:
```bash
# Check Regulus path
ssh root@$BASTION_HOST "ls -la /root/cpt-regulus/run_cpt.sh"

# If missing, re-run Phase 4
make test-regulus-install
```

#### Issue 5: "Test execution failed"
**Common causes**:
- Cluster not accessible (kubeconfig invalid)
- Crucible not working
- Job configuration errors
- Resource constraints (CPU, memory, storage)

**Debug**:
```bash
# Check execution log
cat artifacts/latest/logs/regulus-run.log

# SSH to bastion and debug manually
ssh root@$BASTION_HOST
cd /root/cpt-regulus
source bootstrap.sh

# Test cluster access
export KUBECONFIG=/root/mno/kubeconfig
oc get nodes

# Test Crucible
cd /root/crucible
./bin/crucible --version

# Manually run a simple test
cd /root/cpt-regulus
./run_cpt.sh
```

#### Issue 6: "Results not copied from bastion"
**Cause**: Results directory doesn't exist or rsync/scp failed
**Fix**:
```bash
# Check if results exist on bastion
ssh root@$BASTION_HOST "ls -la /root/cpt-regulus/latest"
ssh root@$BASTION_HOST "readlink -f /root/cpt-regulus/latest"

# Manually copy results
LATEST=$(ssh root@$BASTION_HOST "readlink -f /root/cpt-regulus/latest")
mkdir -p artifacts/manual-copy
rsync -az root@$BASTION_HOST:${LATEST}/ artifacts/manual-copy/
```

### Phase 6 Issues

#### Issue 7: "No run ID found"
**Cause**: Phase 5 didn't complete or state file corrupted
**Fix**:
```bash
# Check state
grep RUN_ID vars/state.env

# If missing, re-run Phase 5
make test-run
```

#### Issue 8: "Validation passed but tests actually failed"
**Cause**: Validation is basic, may not catch all failure modes
**Action**: Always review actual result files:
```bash
# Review result summary
cat artifacts/latest/regulus-results/result-summary.txt

# Review execution log
less artifacts/latest/logs/regulus-run.log

# Check for Crucible errors
grep -i error artifacts/latest/logs/regulus-run.log
```

## Regulus-Specific Details

### Regulus Test Jobs

Regulus organizes tests as **directory paths**, not individual test names.

**Job Directory Structure**:
```
cpt-regulus/
├── 1_GROUP/                       # Basic test group
│   └── NO-PAO/                    # Non-PAO tests
│       ├── 4IP/                   # Basic IP networking
│       ├── 4IP/INTER-NODE/TCP/    # TCP inter-node tests
│       └── ...
└── 2_GROUP/                       # Advanced test group
    └── PAO/                       # Performance Addon Operator tests
        ├── 6IP/                   # IP with PAO
        └── ...
```

**Test Selection**:
- Set `REGULUS_TEST` to directory path, e.g.:
  - `REGULUS_TEST=./1_GROUP/NO-PAO/4IP` - Run 4IP test
  - `REGULUS_TEST=./1_GROUP/NO-PAO/4IP/INTER-NODE/TCP` - Run specific TCP test
  - `REGULUS_TEST=./1_GROUP/NO-PAO/4IP/INTER-NODE/TCP/2-POD` - Run specific 2-POD test

### Auto-Detection: `reg-smart-config`

The `reg-smart-config` script automatically detects:
- Worker node hostnames from cluster
- Available network interfaces
- NIC models and capabilities
- Optimal interface assignments for different test types

**When it works**:
- Standard RHEL/OpenShift clusters
- Common NIC models (Intel, Mellanox)
- Proper node labeling

**When manual config needed**:
- Unusual hardware
- Custom network setup
- Non-standard node naming

### Regulus Environment Files

**`lab.config`**: Infrastructure/environment configuration
- Cluster access (bastion, kubeconfig)
- Worker node hostnames
- Network interface mappings

**`jobs.config`**: Test execution configuration
- Which tests to run (JOBS variable)
- Test parameters (samples, duration)
- OpenShift project to use

**Separation rationale**:
- `lab.config`: Changes when infrastructure changes
- `jobs.config`: Changes when test parameters change
- Allows quick test iteration without reconfiguring infrastructure

## Development Tips

### Interactive Test Selection

**Recommended**: Use the interactive test selection tool to choose tests:

```bash
# From reg-agent root
make configure-tests

# Or from modules/regulus
cd modules/regulus
make configure-tests
```

This provides a menu of common test categories:
- Basic IP tests (4IP variants)
- Performance tests with PAO (6IP variants)
- Custom job paths
- Clear selection (use Regulus defaults)

The tool:
- Shows current test configuration
- Validates job path syntax
- Updates `vars/config.env` with `REGULUS_JOBS`
- Handles space-separated job lists correctly

**Example workflow**:
```bash
# 1. Select tests interactively
make configure-tests
# Choose: "1) 4IP - Basic IP networking"

# 2. Deploy Regulus with selected tests
make -C modules/regulus install

# 3. Run the tests
make -C modules/regulus test

# 4. Change tests and re-run
make configure-tests
# Choose: "3) 4IP/INTER-NODE/TCP/2-POD"
make -C modules/regulus install  # Updates jobs.config
make -C modules/regulus test     # Runs new tests
```

### Manual Test Configuration

Alternatively, you can manually edit configuration:

**Method 1 - Edit config.env**:
```bash
# Edit local config
vi vars/config.env
# Set: REGULUS_JOBS="./1_GROUP/NO-PAO/4IP ./2_GROUP/PAO/6IP"

# Re-run Phase 4 (updates jobs.config on bastion)
make -C modules/regulus install

# Run tests
make -C modules/regulus test
```

**Method 2 - Direct bastion edit** (for quick iteration):
```bash
ssh root@$BASTION_HOST
cd /root/cpt-regulus
vi jobs.config
# Change JOBS line to: export JOBS="./1_GROUP/NO-PAO/4IP"

# Run tests from reg-agent
make -C modules/regulus test
```

**Important**: If you specify `REGULUS_JOBS` in config.env, Phase 4 will overwrite jobs.config. If you leave `REGULUS_JOBS` empty, Regulus defaults are preserved.

### Quick Test Iterations

For rapid testing, use regulus-only mode:
```bash
# Configure once
cat > vars/config.env << EOF
DEPLOY_MODE=regulus-only
BASTION_HOST=my-cluster.example.com
REGULUS_PATH=/root/cpt-regulus
KUBECONFIG_PATH=/root/mno/kubeconfig
REGULUS_TEST=./1_GROUP/NO-PAO/4IP/INTER-NODE/TCP/2-POD
EOF

# Iterate quickly
make run  # Just runs tests, no setup
# Modify test parameters, run again
make run
```

### Debugging Test Failures

1. **Check execution log first**:
   ```bash
   less artifacts/latest/logs/regulus-run.log
   ```

2. **Review result files**:
   ```bash
   ls -lR artifacts/latest/regulus-results/
   cat artifacts/latest/regulus-results/result-summary.txt
   ```

3. **Test manually on bastion**:
   ```bash
   ssh root@$BASTION_HOST
   cd /root/cpt-regulus
   source bootstrap.sh

   # Check configs
   cat lab.config
   cat jobs.config

   # Test cluster access
   export KUBECONFIG=/root/mno/kubeconfig
   oc get nodes

   # Run test manually with verbose output
   ./run_cpt.sh -v
   ```

### Customizing Test Parameters

Common parameters in `jobs.config`:
- `NUM_SAMPLES`: Number of test iterations (default: 3)
- `DURATION`: Test duration in seconds (default: 60)
- `TAG`: Identifier for this test run
- `OCP_PROJECT`: OpenShift namespace to use

Example advanced configuration:
```bash
export JOBS=./1_GROUP/NO-PAO/4IP/INTER-NODE/TCP
export NUM_SAMPLES=10
export DURATION=300
export TAG=TCP-LONG-RUN
export OCP_PROJECT=crucible-regulus
```
